### 2. Backend Separation of Concerns (Folder Structure):
Do not mix UI with business logic. Implement the backend with strict architectural layers:
* `/controllers` – Intercept HTTP requests, validate body/params, call services, and return standardized JSON responses.
* `/services` – Core business logic (e.g., `OrderService` handles database creation AND invokes `ExcelService`).
* `/repositories` – Raw SQL/ORM database query abstraction layer.
* `/validators` – Server-side validation schema (e.g., Zod, Joi, or express-validator) enforcing required fields and phone number formats.
* `/middlewares` – Authentication, role authorization, CORS, CSRF, and error logging.
* `/routes` – Separate `/api/v1/public/*` (customer store) from `/api/v1/admin/*` (protected dashboard).

### 3. Automated Excel Workbook Appending Architecture:
Implement an automated server-side file management utility using a robust library (such as `exceljs` or `xlsx`):
* **File Location:** Maintain a master workbook file at `/exports/orders_master.xlsx`.
* **Headers:** Row 1 must contain exact columns: `Order ID` | `Date/Time` | `Customer Name` | `Phone Number` | `Alternate Phone` | `Complete Address` | `City` | `Ordered Products (Formatted string with quantities)` | `Total Items` | `Subtotal` | `Delivery Charges` | `Additional Charges` | `Grand Total` | `Confirmation Status` | `Admin Notes`.
* **Append Mechanics:** When `OrderService.createOrder()` succeeds, read the workbook, locate the last populated row, append the new order as the next row, apply clean cell formatting, and save the workbook back to disk. Catch file-lock errors gracefully without failing the database transaction.
